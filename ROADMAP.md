# NexusPay 支付编排系统 Roadmap

基于 Hyperswitch 对标分析，列出功能差距及实施优先级。

---

## P0 - 核心能力（必须有）

> 资金安全、交易成功率、合规性基础

### 1. 智能重试引擎（Revenue Recovery）

**目标**：自动恢复失败交易，提升成功率 20-30%

**功能点**：
- [ ] 拒绝码解析与分类
- [ ] 基于错误类型的重试策略
- [ ] 卡 BIN 路由优化
- [ ] 金额分段重试
- [ ] 地区/时间窗口调度
- [ ] 重试预算控制（最大次数、惩罚阈值）
- [ ] 3DS 升级重试

**数据模型**：
```sql
retry_rules        -- 重试规则配置
retry_attempts     -- 重试记录
decline_code_maps  -- 拒绝码映射
```

**工作量**：3-4 周

---

### 2. 对账系统（Reconciliation）

**目标**：确保资金一致性，消除收入泄漏

**功能点**：
- [ ] PSP 交易数据自动拉取
- [ ] 银行结算数据导入
- [ ] 三方对账（内部 + PSP + 银行）
- [ ] 差异检测与报告
- [ ] 手动调整与标记
- [ ] 对账状态流转
- [ ] 历史回溯对账

**数据模型**：
```sql
provider_transactions    -- 渠道原始交易
bank_settlements         -- 银行结算记录
reconciliation_reports   -- 对账报告
reconciliation_items     -- 对账明细
reconciliation_discrepancies  -- 差异记录
```

**工作量**：4-5 周

---

### 3. 渠道健康监控与自动降级

**目标**：实时监控渠道状态，自动故障转移

**功能点**：
- [ ] 成功率实时统计
- [ ] 延迟监控
- [ ] 错误率阈值告警
- [ ] 自动禁用/降级
- [ ] 健康度看板
- [ ] 历史趋势分析

**数据模型**：
```sql
provider_health_metrics  -- 渠道健康指标
provider_outages         -- 渠道故障记录
health_alerts            -- 健康告警
```

**工作量**：2-3 周

---

### 4. 完整 3DS 认证流程

**目标**：合规要求，减少欺诈

**功能点**：
- [ ] 3DS 1.0/2.0 支持
- [ ] 挑战流程处理
- [ ] frictionless 流程
- [ ] 3DS 数据传递
- [ ] 责任转移记录

**数据模型**：
```sql
three_ds_sessions    -- 3DS 会话
three_ds_challenges  -- 挑战记录
```

**工作量**：3-4 周

## P1 - 重要功能（应该有）

> 提升效率、降低成本、增强竞争力

### 5. 网络令牌化（Network Tokenization）

**目标**：授权率提升 3-5%，降低欺诈 26%

**功能点**：
- [ ] Visa/MC/Amex 网络令牌
- [ ] 令牌生命周期管理
- [ ] Cryptogram 生成
- [ ] 令牌刷新/删除
- [ ] PAN 回退机制

**数据模型**：
```sql
network_tokens       -- 网络令牌
token_lifecycle_events  -- 令牌生命周期事件
```

**工作量**：4-5 周（需与卡网络对接）
